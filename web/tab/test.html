    DD Tab Menu (5 styles)

    Author: Dynamic Drive

    Updated Feb 23rd, 08: Adds ability for menu = to revert=20 back to default selected tab when mouse moves out of menu. Only ddtabmenu.js=20 changed!

    Description: DD Tab Menu is a = standards=20 compliant, 2 level tab menu. Move your mouse over a tab, and a 2nd = level=20 content appears beneath it. The script uses CSS to control all of = its=20 appearance, and plain HTML to implement the entire menu tabs and = contents.=20 Highlights of this script are:

    • Standards compliant, with menu tabs and sub contents defined = using=20 plain HTML, making customization a breeze. Five styles to choose = from by=20 default, or customize the tabs yourself.=20
    • Entire menu and its contents search engine friendly.=20
    • Set which menu tab should be selected by default when the = page loads=20 (ie: 1st tab, 2nd tab etc). If the tab contains a sub content, = that=20 content is shown as well.Or, have the script = automatically select=20 a tab when the page loads based on a match between the current = page's=20 URL and one of the menu tabs'. If there is a match, that tab is=20 selected.=20
    • Ability for the selected tab to either remain with the last = tab the=20 user moved his mouse out of, or revert back to the default = selected when=20 mouse moves out of menu. New! =

    Note that the "auto select tab" feature mentioned above is = rudimentary=20 and isn't meant to replace directly selecting or using server side = scripting to identify which tab corresponds to the current page. = It won't=20 account for all possibilities where two URLs are the same. Use it = as a=20 convenience and fallback plan!

    See also: Drop = Down=20 Tabs (5 styles)

    Directions

    To get DD Tab Menu, simply download "ddtabmen= u.zip",=20 which contains all the files that make up this menu and its = accompanying 5=20 styles. For your reference, some files included within the zip are = shown=20 below:

    • ddtabde= mos.htm=20 (demo page showing all 5 Tab Menu instances)=20
    • ddtabmenufiles/ddtabmenu.js=20 (core .js file for DD Tab Menu)=20
    • ddtabmenufiles/ddtabmenu.css=20 (Sample CSS for 1st Tab Menu style)

    Customization

    Setting up and customizing DD Tab Menu is very = simple, as=20 the entire menu consist of regular HTML and styled using CSS. = Logically it=20 makes sense to think of the menu in the following three parts:

    1) Script to call in the HEAD section of your = page to=20 initialize a Tab Menu instance:

    <script = type=3D"text/javascript">
    //SYNTAX:=20 ddtabmenu.definemenu("tab_menu_id", integer OR=20 "auto")
    ddtabmenu.definemenu("ddtabs1", 0) //initialize Tab Menu with = ID=20 "ddtabs1" and select 1st tab by = default
    </script>

    The first parameter (highlighted in red) should be = the ID of=20 the Menu Tab to initialize once the page has fully loaded. The = second=20 parameter lets you set which tab should be selected by default- = enter=20 either an integer (0=3D1st tab, 1=3D2nd tab etc), or = "auto" to=20 have the script try to automatically select the tab that matches = the=20 current page's URL.

    2) HTML for the menu tabs:

    <div id=3D"ddtabs1"=20 class=3D"basictab">
    <ul>
    <li><a=20 href=3D"http://www.dynamicdrive.com" rel=3D"sc1">Home</a></li>
    <li= ><a=20 href=3D"http://www.dynamicdrive.com/new.htm" rel=3D"sc2">DHTML</a></li>
    <l= i><a=20 href=3D"http://www.dynamicdrive.com/style/" rel=3D"sc3">CSS</a></li>
    <li&= gt;<a=20 = href=3D"http://www.dynamicdrive.com/forums/">Forums</a></li&g= t;
    <li><a=20 href=3D"http://tools.dynamicdrive.com/imageoptimizer/">Gif=20 = Optimizer</a></li>
    </ul>
    </div>

    As you can see, the menu tabs is simply an = ordinary HTML=20 list wrapped inside a DIV. Give this DIV a unique ID to be used to = identify it on the page. The arbitrary class = attribute=20 provides the desired style for the tabs. Now, to specify that a = particular=20 tab contain a 2nd level content (shown when mouse rolls over tab), = just=20 insert the code rel=3D"unique_id"=20 inside its <a> element, where "unique_id" is = the ID of=20 the sub content DIV on the page to show (see below). Tabs that = don't carry=20 a "rel" attribute will simple be single level in = nature.

    3) HTML for the 2nd Level Contents

    The 2nd level contents associated with particular = tabs above=20 are defined as individual DIV tags on the page:

    <DIV class=3D"tabcontainer">

<div id=3D"sc1" class=3D"tabcontent">
Sub contents here
</div>

<div id=3D"sc2" class=3D"tabcontent">
Sub contents here
</div>

<div id=3D"sc3" class=3D"tabcontent">
Sub contents here
</div>

</DIV>

    Notice how each DIV carries an arbitrary but = unique ID=20 attribute that matches up with the value of the "rel" = attribute inside the tab menu it is to be associated with. The CSS = class=20 names and outermost DIV are just there to provide styling, and can = be=20 modified/ removed as desired. And that's it!

    Inside ddtabmenu.js

    Near the top of ddtabmenu.js, there are also two = variables=20 you can configure:

    disabletablinks: false, = //Disable=20 hyperlinks in 1st level tabs with sub contents (true or=20 false)?
    snap2original: [true, 300], //Should tab revert back to = default=20 selected when mouse moves out of menu? ([true/false, = delay_millisec]

    The first variable sets whether to disable the = main tabs=20 themselves from navigating to another page when clicked on = (despite them=20 being hyperlinked).

    The second variable sets which tab should remain = selected=20 when the mouse moves out of the menu. False causes = the last=20 tab the user was on before moving out to be selected, while=20 true causes the default selected tab (when the menu = first=20 loaded) to be selected once again. When set to true, = also=20 specify the time delay before the reverting takes place- = practically it=20 should be a number larger than 100.
